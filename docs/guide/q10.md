# 配置 OIDC 认证

::: warning
本文原内容适用于 v0 版本的 OIDC 配置格式，不适用于 v1 及之后版本。
:::

v1 之后 Dashboard 不再使用旧的 `type: oidc`、`oidcIssuer`、`oidcScopes` 等配置字段。当前版本的第三方登录统一使用 OAuth2 绑定配置。

如果你的 OIDC 提供方支持标准 OAuth2/OIDC 端点，请按 [设置 OAuth 2.0 绑定](/guide/q14.html) 中的新版结构填写：

- `client_id`
- `client_secret`
- `endpoint.auth_url`
- `endpoint.token_url`
- `scopes`
- `user_info_url`
- `user_id_path`

配置完成并重启 Dashboard 后，先登录本地账户，在个人信息页绑定第三方账户，再使用登录页的 OAuth2 按钮登录。
